Common Errors & Solutions

Panduan troubleshooting untuk masalah yang sering terjadi

Python & Environment Errors

Problem: Python belum terinstall atau tidak ada di PATH

Solution:

# macOS: Install via Homebrew
brew install python3

# Windows: Download dari python.org
# Centang "Add Python to PATH" saat install

# Verify installation
python3 --version

Problem: Flask belum terinstall atau virtual environment belum diaktifkan

Solution:

# Pastikan virtual environment aktif (ada tulisan (venv))
source venv/bin/activate  # macOS/Linux
venv\Scripts\activate     # Windows

# Install Flask
pip install Flask flask-cors PyMySQL python-dotenv

# Verify
pip list | grep Flask

Problem: Tidak ada permission untuk install global package

Solution:

# JANGAN gunakan sudo!
# Gunakan virtual environment (recommended):
python3 -m venv venv
source venv/bin/activate
pip install flask

# Atau install untuk user saja:
pip install --user flask

Database (MySQL) Errors

Problem: MySQL server tidak berjalan atau credentials salah

Solution:

# Check MySQL running
# macOS:
brew services list

# Start MySQL
brew services start mysql

# Windows: Services ‚Üí MySQL ‚Üí Start

# Test connection
mysql -u root -p

# Check .env file
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=your_password  ‚Üê Pastikan ini benar!
DB_NAME=shop

Problem: Database 'shop' belum dibuat

Solution:

# Login ke MySQL
mysql -u root -p

# Create database
CREATE DATABASE shop;

# Import data
USE shop;
SOURCE Shop.sql;

# Atau via command line
mysql -u root -p shop < Shop.sql

Problem: Password salah atau user tidak ada

Solution:

# Reset MySQL password (macOS)
mysql.server stop
mysqld_safe --skip-grant-tables &
mysql -u root

# Di MySQL prompt:
FLUSH PRIVILEGES;
ALTER USER 'root'@'localhost' IDENTIFIED BY 'new_password';
quit;

# Restart MySQL
mysql.server restart

# Update .env file dengan password baru

Flask Application Errors

Problem: Port 5000 sudah digunakan process lain

Solution:

# Option 1: Kill process di port 5000
# macOS/Linux:
lsof -ti:5000 | xargs kill -9

# Windows:
netstat -ano | findstr :5000
taskkill /PID [PID_NUMBER] /F

# Option 2: Ganti port di app.py
app.run(debug=True, port=5001)  # Ganti ke 5001

Problem: Route tidak terdaftar atau URL salah

Solution:

# Check:
1. Blueprint registered di app.py?
   app.register_blueprint(customers_bp, url_prefix='/api/customers')

2. URL benar?
   ‚úÖ http://127.0.0.1:5000/api/customers/
   ‚ùå http://127.0.0.1:5000/customers/

3. Flask server running?
   python app.py

4. Check available routes:
   flask routes  # Di terminal

Frontend & CORS Errors

Problem: Browser block request karena CORS policy

Solution:

# Install flask-cors
pip install flask-cors

# Di app.py:
from flask_cors import CORS

app = Flask(__name__)
CORS(app)  # Enable CORS untuk semua routes

# Restart Flask server!
# Ctrl+C lalu python app.py

Problem: Frontend tidak bisa connect ke backend

Solution:

# Check:
1. Flask server running?
   python app.py

2. URL benar di JavaScript?
   const API_URL = 'http://127.0.0.1:5000/api';

3. Test di browser langsung:
   http://127.0.0.1:5000/api/customers/

4. Check browser console (F12) untuk detail error

Problem: JavaScript error atau API tidak return data

Solution:

# Debug steps:
1. Buka Browser Console (F12)
2. Check ada error JavaScript?
3. Check Network tab:
   - Request sent?
   - Response status 200?
   - Response body ada data?

4. Test API di Postman:
   GET http://127.0.0.1:5000/api/customers/

5. Add console.log() di JavaScript:
   console.log(data);  // Lihat data yang diterima

6. Check element ID benar:
   getElementById('customersTable')  // ID match?

General Debugging Tips

Terminal Tips
  • Baca error message dengan teliti
  • Copy error message ke Google
  • Check line number yang error
  • Gunakan print() untuk debug
Browser Tips
  • Selalu buka Console (F12)
  • Check Network tab untuk HTTP requests
  • Clear cache jika ada masalah CSS/JS
  • Use Incognito mode untuk test
General Tips
  • Restart Flask server setelah edit code
  • Refresh browser setelah edit JS/CSS
  • Check typo di variable names
  • Pastikan indentation benar (Python)
